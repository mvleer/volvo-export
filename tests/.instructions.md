# Test Strategy — Volvo Trip Pipeline

## Philosophy
This is a data pipeline. The enemy is silent corruption: a bug that produces
a plausible-looking but wrong output. Every test layer must be independently
runnable and independently meaningful.

## Test pyramid

```
        [ e2e ]          1 test module: run full pipeline on fixture exports,
                          assert per-year output files are correct end-to-end.
                          Slow-marked; excluded from default pytest run.

     [ output quality ]  Per-year file tests. Strict. These are the files
                          that get used for analysis — they must be perfect.
                          Parametrized over discovered volvo-trips-YYYY.csv files.

  [ merge / dedup ]      Cross-file consistency. Runs on the merged dataset
                          before split. Catches overlap handling and sort order.
                          Uses synthetic fixtures only.

[ raw input integrity ]  Fast, fail-early. Reject bad exports before anything
                          else runs. Parametrized over real raw/ files.
```

## Fixture rules

- **Never use real data files as test fixtures.** All synthetic fixture data is
  defined in `tests/conftest.py`.
- Fixtures must cover **edge cases by construction**:
  - A trip that spans midnight (Started in one day, Stopped in the next)
  - A trip with a decimal distance (e.g. `125,3 km` raw → `12,53` output)
  - A trip present in two overlapping exports (the dedup target)
  - A trip at a year boundary (Dec 31)
  - A trip with zero fuel consumption
- Use `pytest` fixtures with `tmp_path` for all file I/O in synthetic tests.
  No test may write to the real `raw/` or output directories.
- Shared fixtures live in `conftest.py`. No fixture may be defined inside a
  test file — if it is needed by more than one test, it belongs in conftest.

## Layer contracts

### Layer 1 — raw input (`test_raw_inputs.py`)
- Scope: each file in the real `raw/` directory, parametrized by file.
- Must not import or depend on pipeline code.
- Validates: parseable as semicolon-delimited CSV; required columns present;
  every `Started`/`Stopped` value matches `DD/MM/YYYY HH:MM`; no completely
  empty rows.
- A missing or empty `raw/` directory is itself a test failure.
- **Failure here signals that a new export is malformed — fix the export
  before running the pipeline.**

### Layer 2 — merge & dedup (`test_merge.py`)
- Scope: the merged+deduplicated dataset produced internally by the pipeline,
  exercised via the `merged_rows` and `output_dir` fixtures.
- Natural key for dedup: `(Started, Start odometer (km))` — both after ISO
  date conversion.
- Must assert: `len(unique keys across all raw rows) == len(merged rows)`.
- Must assert: output is sorted ascending by `Started` (ISO string sort).
- Must assert: no trip's natural key appears in more than one year-split file.

### Layer 3 — output quality (`test_output.py`)
- Scope: each real `volvo-trips-YYYY.csv` file in the project root,
  parametrized dynamically so adding a new year file extends coverage
  automatically.
- Every assertion is a **separate test function**, not a long assert chain,
  so failures are individually identifiable.
- Required assertions per year-file:
  - Header is exactly `CANONICAL_HEADER`, in order.
  - All rows have `len(row) == len(header)` (no ragged rows).
  - `Started` and `Stopped` match `\d{4}-\d{2}-\d{2} \d{2}:\d{2}`.
  - `Distance (km)` and `Fuel consumption (l)` contain no alpha characters.
  - All numeric fields parse as float after replacing `,` with `.`.
  - All numeric fields are ≥ 0.
  - `Distance (km)` < 2000 for every row (sentinel for missing ÷10 correction).
  - `Year` == `f"{year}-{month}"` derived from `Stopped`, no leading zero on month.
  - All `Stopped` dates belong to the file's stated year (year boundary check).
  - EU comma notation: no `.` in numeric fields that contain a decimal separator.

### e2e (`test_pipeline_e2e.py`)
- Marked `@pytest.mark.slow`; excluded from the default `pytest` run.
- Run explicitly with: `pytest -m slow`
- Runs `run_pipeline()` via the `output_dir` fixture (tmp_path-based).
- Also runs the script as a subprocess via `--raw-dir`/`--output-dir` CLI args.
- Spot-checks three known rows from the fixture data.
- Asserts total output row count == unique raw natural keys (dedup count check).

## Rules for writing new tests

1. **A new transformation in the pipeline requires a new Layer 3 assertion**
   before the work is considered complete.
2. **A bug fix must produce a regression fixture row** that would have triggered
   the original bug, plus a test that would have caught it.
3. **Do not assert on row count** unless testing dedup specifically. Row count
   changes as new exports are added; structural invariants do not.
4. **Do not read from `volvo-trips-cleaned.csv`** — that file is a legacy
   artifact. Tests target per-year files only.
5. Parametrize over output year files dynamically (discovered from the project
   root at collection time) so a new year file automatically enters coverage.
